Найпростіша і найсоціальніша платіжна система

Для прийняття платежів в системі Okcash досить просто дати свою адресу або показати QR-код. Така простота робить систему доступною для всіх користувачів і задовольняє потреби більшості клієнтів. Прозорість системи і легкість використання сприяють пожертвам, чайових і заохочень. Ви навіть можете поділитися монетами Okcash через IRC або соціальні мережі, такі як Reddit або Twitter при цьому не встановлюючи гаманець.

API від третіх сторін

Існує багато платіжних сервісів від третіх сторін, що надають API; вам не потрібно самому зберігати Okcash на сервері і дбати про безпеку. Крім цього, більшість цих API дозволяють обробляти замовлення і конвертувати Okcash в місцевій валюті за вигідним курсом. Завдяки відкритому вихідному коду їх можна також використовувати для створення нових сервісів.

Ви можете стати фінансовою системою для самого себе

Якщо ви не хочете використовувати API від третіх сторін, ви можете інтегрувати вузол Okcash в свої додатки; це дозволить вам стати самому собі і банком, і платіжним оператором. Несучи всю повноту відповідальності, ви зможете створювати вражаючі системи, які будуть обробляти транзакції Okcash як ви побажаєте.

Використання адрес Okcash для відстеження замовлень

Okcash створює унікальну адресу для кожної транзакції, тому, якщо ви створюєте платіжну систему для обробки замовлень, вам просто потрібно генерувати і моніторити адреси OK для кожного платежу. Адреси Okcash слід використовувати лише один раз для однієї транзакції.

Більшість рішень безпеки знаходиться на стороні клієнта

Більшість рішень безпеки знаходиться в протоколі, що дозволяє позбутися від стандарту PCI. Захист від шахрайства зводиться до моніторингу кількості підтверджень . Для безпечного зберігання валюти OK потрібно тільки убезпечити свій гаманець використовувати HTTPS або інші безпечні протоколи для відправки запитів платежів своїм покупцям.

Майбутнє соціальних платежів

Okcash дозволяє створювати нові і креативні онлайн-сервіси, які не могли існувати раніше через технологічні обмеження. Сюди входять: система чайових, автоматичні платежі, розподілені сервіси краудфангдінга, управління платежами в часі, публічне відстеження коштів, низькоризикові послуги гарантів, канали мікроплатежів, пули для стейкінга і багато іншого.
